# Governance

## Model

PortAtlas uses founder-led governance during its pre-implementation and early release stages. The founder accepts scope gates, appoints maintainers, and resolves decisions that affect public naming, security posture, compatibility, or release readiness. Maintainers may review and merge changes within accepted requirements and ADRs.

## Decision process

- Product scope is controlled by the charter, BRD, PRD, requirements, and roadmap.
- Material technical or policy choices use numbered Architecture Decision Records.
- Security and privacy changes require explicit review against the threat model.
- A release advances only when its documented gate is satisfied with evidence.
- Disagreement should be resolved with written alternatives, consequences, and reversible experiments where possible.

## Contributions and provenance

Contributors sign off commits under the Developer Certificate of Origin. There is no relicensing contributor license agreement. Accepted contributions are Apache-2.0 licensed.

Apache-2.0 permits commercial use without payment. Sponsorship is optional, does not buy usage permission, and does not change technical or moderation decisions. A future funding configuration requires an active founder-supplied sponsorship handle.

## Name and marks

`PortAtlas` is a working title. The Apache-2.0 copyright license does not grant trademark rights. Public naming, package namespaces, domains, and artwork remain blocked by [ADR-0023](docs/adr/0023-working-name-collision-and-trademark-clearance.md).

## Changes to governance

Governance changes require a pull request explaining the motivation, affected stakeholders, transition plan, and founder approval during the founder-led phase.
